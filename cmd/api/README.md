## api（HTTP/JSON 对外接口）

### 做什么

- **对接前端**：提供后台管理系统对外 HTTP/JSON API（列表/详情/导出等），严格参数校验与排序白名单。
- **读写存储**：查询 MySQL/MongoDB；按需要使用 Redis 做首屏/默认列表缓存（cache-first），用户主动查询走 DB（db-first）。
- **鉴权与授权**：接收认证侧透传的身份标识（如 `admin_id`/token claims），在 API 层执行 RBAC 权限校验。
- **后台任务执行器**：在 API 进程内执行导出任务、缓存刷新、过期清理等（多实例用分布式锁避免重复执行）。
- **管理操作上链**：封装并调用 `API_TX` 广播链上交易（如上/下架、参数更新等，后续实现）。

### 不做什么

- **不消费 Kafka**：不承担消费与落库职责。
- **不做链上采集**：不负责扫链与事件规范化。
